Bad command or file name은MS-DOS에서 유효하지 않은 명령을 내릴 경우에 나타나는 오류 메시지이다.구문 오류메시지이며1980년대베이직인터프리터가 그 기원이다.COMMAND.COM파일에 내장된 내부 명령어가 아니면서 디렉터리에 해당 실행파일이 존재하지 않을 경우[1]이 메시지가 출력된다.C:₩&gt;abcBad command or file name.C:₩&gt;_위의 예는 COMMAND.COM에 ABC라는 내부 명령어가 내장되어 있지 않고 또 ABC.COM, ABC.EXE, ABC.BAT 셋 중 어느 하나도 C:\에 없음을 의미한다.명령 또는 파일 이름이 올바르지 않습니다.명령 또는 파일 이름이 틀립니다.&lt;입력 문자열&gt; is not recognized as an internal or external command, operable program or batch file.&lt;입력 문자열&gt; 은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.이렇게 메시지가 길어진 까닭은 초보자들이 단순히 "명령 또는 파일 이름이 잘못되었다"는 메시지를 잘 이해하지 못하였기 때문이다.
